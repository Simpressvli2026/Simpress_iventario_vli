1. Netlify — Hospedagem e Backend Serverless

O que é

Plataforma de hospedagem frontend (React/Vite) com suporte a serverless functions. O código é versionado no GitHub e o deploy é automático a cada commit. O backend (API) roda como Netlify Functions sem necessidade de gerenciar servidores.

Segurança e Certificações

  • SOC 2 Type II Relatório de auditoria independente
  • ISO 27001 Gestão de segurança da informação
  • PCI DSS Compatível com padrão de cartões
  • GDPR e CCPA Privacidade de dados
  • AES-256 Dados criptografados em repouso
  • TLS 1.2+ Tráfego criptografado em trânsito
  • WAF Web Application Firewall com regras OWASP Top 10
  • DDoS automático, rate limiting, firewall por IP/país

Clientes de grande porte

Nike Google Spotify Netflix McDonald's Facebook

Avaliação para o projeto

Recomendado O plano Free cobre perfeitamente o projeto (100 GB de banda/mês, 300 horas de build/mês). Para escala futura, planos Pro e Enterprise oferecem SLA, SSO, e suporte dedicado.

2. Turso (libSQL) — Banco de Dados SQL na Nuvem

O que é

Banco de dados SQL compatível com SQLite, mas rodando na nuvem. Utilizamos o pacote @libsql/client (libSQL) que é um fork do SQLite com extensões, considerado production-ready. O novo Turso Database (reescrita em Rust) ainda está em beta, mas não é o que estamos usando.

Segurança

  • SOC 2 Certificação em andamento para Turso Cloud
  • Criptografia em repouso Todos os dados criptografados a nível de volume
  • BYOK Bring Your Own Key (você controla a chave de criptografia)
  • AEAD Algoritmos de criptografia autenticada (AES-GCM, ChaCha20)
  • Backups contínuos e Point-in-Time Recovery

Avaliação para o projeto

Recomendado O libSQL (que usamos via @libsql/client) é maduro e usado em produção. Para 2700 registros é mais que suficiente. A Turso Cloud tem plano gratuito generoso e escala facilmente.

Observação importante

O Turso Database (reescrita em Rust) ainda está em beta e não recomendado para produção. Nós usamos o libSQL (fork do SQLite), que é production-ready.

3. Cloudinary — Gerenciamento de Imagens

O que é

Plataforma completa de gerenciamento de imagens e vídeos: upload, armazenamento, transformação sob demanda e entrega via CDN global. Usamos para armazenar as 3 fotos de cada notebook registrado.

Segurança e Certificações

  • SOC 2 Type II Auditado pela Deloitte anualmente
  • ISO 27001 Gestão de segurança
  • HIPAA Conformidade para dados de saúde
  • GDPR Conformidade com privacidade europeia
  • Criptografia Em repouso e em trânsito
  • Bug Bounty Programa de recompensa por vulnerabilidades (24/7)
  • Testes de penetração anuais por terceiros
  • Tokens assinados, ACL por IP/país, SSO, MFA

Clientes de grande porte

Netflix Airbnb Etsy Shopify BuzzFeed NASA

Avaliação para o projeto

Recomendado Plano Free atende bem (uploads via unsigned preset). Para produção mais robusta, planos pagos oferecem tokens de acesso, CDN dedicada e suporte Enterprise.

Veredito

O Power Apps seria mais rápido de prototipar, mas traria custos recorrentes por usuário, exigiria login Microsoft (barreira para os funcionários) e nos prenderia ao ecossistema Microsoft.

7. Conclusão

As três plataformas são seguras, maduras e amplamente adotadas por empresas de grande porte.

  • Netlify — Padrão de mercado para frontend serverless. Usado por gigantes como Nike e Google.
  • Turso/libSQL — Banco SQL moderno e escalável. O libSQL é production-ready; o novo Turso DB está em beta.
  • Cloudinary — Praticamente padrão da indústria para gestão de imagens. Presente em milhares de empresas globais.

Para este projeto — 2700 registros, fotos de notebooks, formulário público — todas as três plataformas operam muito folgadas nos planos gratuitos. Caso haja crescimento, todas oferecem planos pagos com suporte Enterprise, SLA e recursos avançados de segurança.
